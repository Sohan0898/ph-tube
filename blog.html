1. Discuss the scope of var, let, and const

Ans : var, let, and const are three different variable declaration keywords in JavaScript, each with its own scope and behavior.They were introduced to manage variable declarations and scoping in a more structured way. Let's discuss the scope of each of these keywords:

1. var Scope: Variables declared with var are function-scoped or globally scoped, depending on whether they are declared inside a function or in the global scope. If a variable is declared using var within a function, it is accessible within that function and any nested functions. However, variables declared with var outside of any function block are globally scoped and accessible throughout the entire JavaScript program. var variables are subject to hoisting, which means they are moved to the top of their scope during compilation. This can lead to unexpected behavior if not understood properly.

2. let Scope: Variables declared with let are block-scoped, meaning they are limited in scope to the block, statement, or expression where they are defined. 2. Tell us the use cases of null and undefined

Ans : In JavaScript, both null and undefined represent the absence of a value, but they are used in slightly different contexts. Let's explore the use cases of each:

1. null: null is a value that represents the intentional absence of any object value or a deliberate assignment of "no value." It is often used to explicitly indicate that a variable should hold no value or that a value is missing. It can be assigned to variables to indicate that they have no valid object or value assigned to them.

2. undefined: undefined indicates that a variable has been declared but has not been assigned any value. It is the default value for uninitialized variables. When a variable is declared but not assigned a value, its value is automatically set to undefined. Functions that do not explicitly return a value also return undefined.

It's important to note that while both null and undefined can indicate the absence of a value, they are not interchangeable in all situations. JavaScript treats them as distinct values, and their use cases have subtle differences.

3. What do you mean by REST API?

Ans : A REST API, or Representational State Transfer Application Programming Interface, is a way for software applications to communicate over the internet. It follows a set of rules that make interactions between clients (like apps or websites) and servers (where data is stored) consistent and efficient. REST APIs use standard HTTP methods like GET, POST, PUT, and DELETE to perform actions on resources (data entities) identified by unique URLs. This approach allows different software systems to exchange data in a structured and scalable manner.
